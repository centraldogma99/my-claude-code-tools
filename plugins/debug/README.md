# debug

이슈의 근본 원인을 추적하여 디버깅하는 워크플로우 스킬

## 워크플로우

1. **증상 파악** — 에러 메시지, 스택 트레이스, 재현 조건 수집
2. **역추적** — 증상 지점에서 호출 체인을 거슬러 올라가며 소스 코드 직접 확인
3. **인과 관계 체인 정리** — 근본 원인 → 전파 경로 → 증상까지의 흐름을 정리
4. **소스 수준 수정** — 소비자 측 우회가 아닌 근본 원인 코드를 직접 수정

## 핵심 원칙

- 관련 코드의 동작 방식을 **추측하지 않고 직접 읽는다**
- 호출하는 쪽의 우회가 아닌 **소스 수준에서 수정**한다
- 수정 전에 반드시 **인과 관계 체인**을 먼저 정리한다
