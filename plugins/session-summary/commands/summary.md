# Session Summary

특정 기간의 Claude Code 세션들을 **모든 프로젝트 디렉토리**에서 수집하여 요약한다.

## Workflow

### 1. 날짜 범위 결정

사용자 요청에서 날짜 범위를 파악한다.

| 요청 예시 | start | end |
|---|---|---|
| "어제 세션 정리해줘" | 어제 날짜 | 어제 날짜 |
| "이번 주 작업 요약" | 이번 주 월요일 | 오늘 |
| "3월 1일~5일" | 2026-03-01 | 2026-03-05 |
| "최근 세션 정리" | 어제 | 오늘 |

### 2. 세션 데이터 추출

`scripts/extract_sessions.py` 실행으로 raw 데이터를 수집한다.

```bash
python3 {PLUGIN_DIR}/scripts/extract_sessions.py --start YYYY-MM-DD [--end YYYY-MM-DD]
```

- `--end` 생략 시 `--start`와 동일한 날짜로 처리
- 출력: JSON (프로젝트별 세션 목록, 각 세션의 사용자 메시지 포함)

**중요**: 스크립트는 `~/.claude/projects/` 하위의 **모든** 프로젝트 디렉토리를 탐색한다.
worktree, 다른 레포지토리 등 프로젝트별로 세션 파일이 분리 저장되므로 반드시 전체를 탐색해야 누락이 없다.

### 3. 분석 및 요약

추출된 JSON을 분석하여 아래 원칙에 따라 정리한다.

**요약 원칙**:
- 프로젝트/worktree 단위로 그룹핑
- 각 그룹 내에서 관련 세션들을 하나의 작업 주제로 묶기 (예: 같은 기능에 대한 여러 시도)
- 각 작업의 핵심 내용을 1~3줄로 요약
- 세션 수가 많은 작업일수록 비중 있게 설명

**출력 형식**:

```
## YYYY-MM-DD (~ YYYY-MM-DD) 세션 정리

### 1. [작업 주제] (프로젝트명)
N개 세션

- 핵심 작업 내용
- 주요 의사결정이나 결과물

### 2. [작업 주제] (프로젝트명)
...
```

### 4. 보충 (선택)

필요시 git log로 해당 기간 커밋/PR과 세션을 연결한다.

```bash
git log --oneline --after="YYYY-MM-DD" --before="YYYY-MM-DD" --all
```
