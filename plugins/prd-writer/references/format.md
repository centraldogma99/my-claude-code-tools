# PRD 표준 포맷

## 필수 섹션 (순서대로)

### 1. 헤더
| 항목 | 내용 |
|------|------|
| 작성일 | YYYY-MM-DD |
| 상태 | Draft / Review / Approved |
| 작성자 | 이름 |
| 이해관계자 | 관련 팀/사람 |

### 2. 개요
- **배경**: 현재 상황과 문제가 왜 존재하는지
- **문제 정의**: 구체적으로 무엇이 문제인지 (2~3문장)

### 3. 목표
- ID가 부여된 목표 목록 (G1, G2, ...)
- 각 목표는 **사용자 관점**에서 서술 ("~할 수 있도록 한다")
- **비목표 (Out of Scope)**: 이 PRD에서 다루지 않는 것 명시

### 4. 대상 사용자
- 페르소나별 역할 + 핵심 니즈 테이블

### 5. 사용자 시나리오
- 번호가 매겨진 단계별 시나리오 (1, 2, 3...)
- 주요 플로우별로 하나씩
- 사용자 행동 중심 서술 ("담당자가 ~를 클릭한다")

### 6. 기능 요구사항
- 목표별로 그룹핑
- 우선순위 표기: P0 (필수), P1 (중요), P2 (권장)
- 각 항목은 한 문장으로 검증 가능하게

### 7. 비즈니스 규칙
- ID가 부여된 규칙 (BR1, BR2, ...)
- 시스템이 따라야 하는 정책/조건

### 8. 성공 지표
- 측정 가능한 지표 + 측정 방법 + 목표값

### 9. 제약사항 및 의존성
- 외부 의존성 (API, 서비스 등)
- 기술적 제약

### 10. 변경 이력
- 버전, 날짜, 변경 내용
