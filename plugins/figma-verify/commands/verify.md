# Figma 디자인 검증 워크플로우

마크업 작업이 완료된 상태에서, 구현 결과물이 Figma 디자인과 일치하는지 검증하고 수정한다.

## 절차

### 1. 디자인 스크린샷 확보

Figma MCP 도구(`get_screenshot` 또는 `get_design_context`)를 사용하여 원본 디자인 스크린샷을 확보한다.
이미 대화 컨텍스트에 Figma 스크린샷이 있으면 그것을 사용한다.

### 2. 구현 결과물 스크린샷 촬영

`agent-browser` 스킬을 사용하여 개발 서버에서 구현된 페이지의 스크린샷을 촬영한다.

### 3. 비교 및 차이점 식별

두 스크린샷을 비교하여 다음 항목의 차이점을 식별한다:

- 레이아웃 및 간격 (margin, padding, gap)
- 색상 (배경, 텍스트, 보더)
- 폰트 (크기, 굵기, 행간)
- 컴포넌트 크기 및 비율
- 누락된 요소

### 4. 수정

식별된 차이점을 하나씩 수정한다.
수정 후 다시 스크린샷을 촬영하여 해당 차이가 해소되었는지 확인한다.
차이가 남아있으면 3~4단계를 반복한다.

### 5. 완료 보고

모든 차이점이 해소되면 사용자에게 보고한다:

- 최종 스크린샷
- 발견 및 수정한 차이점 목록
- 수정 완료 확인
